HannaAndersson.com:

the merchant website of the US clothes store

A renowned clothes store selling mainly pajamas and baby/children clothing.

My project: study the entire Sale section of the website

I spent the most time on the web-scraping phase and I found it painstakingly difficult.

- Started with a "403 response" ("Access forbidden").

- In the website you can only upload 12 new products with every click. Hence in order to scrape 200 products I had to loop over 18 webpages.

- Then for some items the html tags are missing. For example if a product is sold out there is no longer a html pricing tag, but there is a specific sold-out tag. Same for colors: if the product exists in other colors there is a color tag, but there is no color tag for the products which are available in one color only.

- At  the end of the web-scraping phase my 8 arrays varied from 204 to 216 values!

The project mainly made me review Python loops and Pandas dataframes.

Some take-aways from data analysis

- Around 200 items on sale on the website at the moment.

- A high discount rate: sale price is 40% of regular price at median.

- Most products (2/3) have not been rated by consumers. Of those rated, near all are rated 4.5/5 (52%) or 5/5 (48%).

- Median price on sale is $17 and average price is $17,65  --which means that there are not many extreme values. Just one item at  $58 on sale draws the average up.

Improvements

I really should have spent less time on the web-scraping process and more time on the data analysis. Perhaps if I had been able to use regex instead of CSS Selector?

I would have loved looking more into the data, to understand which of boys or girls clothes are discounted the most, which categories of clothes are on sale and if there is a rationale behind the sale figures.

Had I had more time I could also have used mySQL for sophisticated queries.

If I were to start from scratch...

I would chose another website, easier to scrape under limited time conditions!
